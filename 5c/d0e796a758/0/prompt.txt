## Step 1: Add 5 generator functions to `eval-generator.ts` (both copies)

Append these exported functions to the end of both eval-generator files:

### `generateVitestConfig()` → `vitest.config.ts`
- Node environment, `src/**/*.test.ts` include pattern
- v8 coverage targeting `src/eval/**/*.ts`, excluding test files and handlers

### `generateReportGeneratorTests()` → `src/eval/__tests__/report-generator.test.ts`
Tests for the pure-function `ReportGenerator` class:
- `generateJSON` — all-pass → `overallStatus: 'pass'`, mixed → `'mixed'`, all-fail → `'fail'`
- Correct `totalDurationMs` and cost aggregation
- Baseline comparison delta computation
- `generateMarkdown` — correct headers, table structure, cost section

### `generateTaskRegistryTests()` → `src/eval/__tests__/task-registry.test.ts`
Tests for the in-memory `TaskRegistry`:
- `register()` / `get()` / `has()` / `listTypes()` behavior
- `createDefaultRegistry()` includes all 5 built-in handler types
- Handler `execute()` returns error result when no command is provided (sync path, no subprocess needed)

### `generateSpecParserTests()` → `src/eval/__tests__/spec-parser.test.ts`
Tests for `SpecParser` using temp files (`fs.mkdtempSync`):
- Parse valid YAML and JSON specs
- Throws on non-existent file, unsupported extension
- Validation rejects missing required fields, empty tasks, tasks without id/type

### `generateResultsManagerTests()` → `src/eval/__tests__/results-manager.test.ts`
Tests for `ResultsManager` using a temp base directory:
- `createRun()` returns `run-*` id and creates subdirectories
- Manifest and task result write/read roundtrips
- `readManifest` / `readTaskResult` return null for missing entries
- `readAllTaskResults` returns sorted array
- `writeSummary` creates both `.json` and `.md` files
- `listRuns` returns runs newest-first
